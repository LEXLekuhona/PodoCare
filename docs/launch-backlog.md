Задачи **вне репозитория** или требующие решения бизнеса/юристов/облака. Входная точка «до коммерческого»: `docs/pre-commercial-readiness.md`. Процедуры на каждый релиз: `docs/release-checklist.md`. Инфраструктура и smoke: `docs/infrastructure-and-operations.md`, `docs/reminders-lifecycle-runbook.md`. Статус фич: `docs/roadmap.md`.

---

## 1. Репозиторий и процесс

- [ ] Свести изменения в `main` через PR (ревью, зелёный CI).
- [x] **Автодеплой staging/VPS** после зелёного CI на `main`: `.github/workflows/deploy-staging.yml`, `scripts/deploy-staging.sh` (секреты `STAGING_SSH_*` в GitHub Actions).
- [ ] На ветке релиз-кандидата **отметить** выполнение `docs/release-checklist.md` после реального прогона (включая smoke на staging, когда среда будет готова).

---

## 2. Staging и production

- [ ] **Staging**, близкий к prod: отдельные Postgres, Redis, S3-совместимое хранилище, секреты вне git — сверка с разделом 1 в `docs/infrastructure-and-operations.md` и подразделом «Сверка с этим бэклогом» там же.
- [ ] **Production**: managed Postgres (бэкапы, по возможности PITR), Redis, object storage, секреты только во внешнем хранилище — сверка с разделом 2 в `docs/infrastructure-and-operations.md` и тем же подразделом.
- [ ] **Алерты** по `GET /api/v1/health/queues` на staging и prod (`ALERT_*`, раздел 5 в `docs/infrastructure-and-operations.md`; ненулевой exit для внешней синтетики — `scripts/health-queues-synthetic-check.sh`).

## 5. Юридика и ПДн (РФ)

Минимум для **выбранной** модели (B2C vs B2B vs gos) — письменно с юристом; тексты опубликованы, ссылки в вебе, мобильном клиенте и чек‑ауте согласованы с §3 (сторы) и платежным блоком §4.

- [ ] **152-ФЗ / ПДн:** политика конфиденциальности; согласие(я) на обработку; цели, сроки и основания; права субъекта и порядок отзыва; субоператоры/обработчики по факту интеграций; трансграничная передача — если применимо.
- [ ] **Медданные / особые категории:** отдельное правовое оформление (согласие, усиленные меры, доступ) — если юрист квалифицирует данные в продукте таким образом.
- [ ] **B2C:** пользовательское соглашение / публичная оферта (кто оператор, предмет услуг, оплата, поддержка, возвраты); согласованность с эквайрингом и текстами в приложении.
- [ ] **B2B (студия ↔ платформа):** договорная рамка или оферта для студий; распределение ролей оператора/субоператора ПДн; инциденты и уведомления; выгрузка/удаление данных при расторжении — по необходимости.
- [ ] **Лицензия на ПО:** если модель продажи требует — лицензионные условия, ограничения использования, обновления (с юристом).
- [ ] **B2B / gos (отдельный трек, не блокирует B2C):** реестр отечественного ПО (сроки и бюджет внесения при выбранной цели); требования заказчика (ИБ, хостинг, аттестация и т.п.) — оценка и план отдельно от быстрого коммерческого B2C.

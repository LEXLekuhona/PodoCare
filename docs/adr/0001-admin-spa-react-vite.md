# ADR 0001: Админ-панель как отдельное React SPA (Vite)

**Статус:** принято  
**Дата:** 2026-05-02

## Контекст

В монорепозитории уже есть клиентское приложение на Expo/React Native и API на NestJS. Операционные данные и контент правятся через защищённые эндпоинты (`admin/catalog`, `admin/education` и др.). Нужен явный способ разрабатывать и деплоить интерфейс для персонала без смешения с мобильным клиентом.

## Решение

- Веб-админка живёт в **`apps/admin`**: **React + TypeScript + Vite**.
- Авторизация персонала: **`POST /auth/staff/login`**, хранение пары токенов и профиля на клиенте (например `localStorage`), при необходимости — обновление через **`POST /auth/refresh`**.
- Общие перечисления и типы по возможности брать из **`@podocare/shared-types`**.
- Публичный маркетинговый сайт и SEO-требования **не** реализуются в этом приложении; при появлении — отдельный проект (например Next.js или статический сайт).

## Последствия

- Положительные: быстрый локальный DX (Vite), привычный стек для таблиц/форм, независимые релизы от мобильного приложения.
- Отрицательные: второй фронтовый рантайм (рядом с RN); нужно следить за CORS (`API_CORS_ORIGINS`) и версией префикса API (`VITE_API_URL`, например `/api/v1`).

## Альтернативы (отклонены)

- **Expo Web для админки** — возможно при желании одной кодовой базы, но для операционных панелей обычно хуже по UX разработки и экосистеме таблиц/форм.
- **Встраивание админки в Next без необходимости SSR** — избыточно до появления публичных SEO-страниц в том же приложении.

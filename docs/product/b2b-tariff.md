# Тариф Podocare (B2B SaaS) — черновик v0.1

**Статус:** для согласования с бизнесом перед офертой и продажами.  
**Модель:** подписка студии/сети; мобильное приложение white-label; публикация в сторах — [трек A или B](customer-release-checklist.md).

---

## Для кого

Независимые подологические студии и небольшие сети (1–N точек), которым нужны:

- записи, напоминания, кабинет специалиста;
- контент-воронка, квиз, программы, оплаты (по мере включения модулей);
- **брендированное** мобильное приложение для клиентов.

**Оператор ПДн клиентов студии** — студия (заказчик). **Podocare** — лицензиар ПО и обработчик по поручению (см. договор).

## Что входит в подписку (все платные планы)

1. **Изолированный tenant** в общем облаке (`networkId`): студии, персонал, клиенты, записи, уведомления.
2. **Веб-админка** для `NETWORK_OWNER`, `STUDIO_ADMIN`, `SPECIALIST` (планшет в кабинете).
3. **Сборка мобильного приложения** под бренд заказчика (имя, иконка, bundle id, привязка к API tenant).
4. **Обновления платформы** (исправления, согласованные минорные релизы) — пока активна подписка.
5. **Резервное копирование БД** на уровне инфраструктуры prod (когда prod поднят по `launch-backlog` §2).

## Trial (пилот)

- **Срок:** 14–90 дней (фиксируется в договоре).
- **Объём:** 1 студия, до 3 специалистов, трек B или ограниченный трек A.
- **После trial:** переход на «Старт»/«Сеть» или отключение tenant (`network` неактивна, данные по политике хранения в договоре).

---

## Не входит в подписку

- Разработка **индивидуальных** фич под одного заказчика (отдельная оценка).
- Юридические тексты (политика ПДн, оферта B2C для клиентов студии) — **заказчик** + ревью юриста; мы даём **шаблоны**.
- Маркетинг, ведение стора, ASO, контент-стратегия.
- Гарантия прохождения ревью App Store / Google Play при нарушении правил стора или неполных материалах заказчика.

## Оплата и расторжение

- **Период:** календарный месяц, предоплата (или квартал со скидкой **___ %**).
- **Просрочка:** напоминание → приостановка доступа к админке/tenant через **___** дн. → расторжение по договору.
- **Расторжение:** экспорт данных по запросу (формат и срок — в B2B-договоре); приложение в сторах — по треку (снятие listing ответственность заказчика).

## Что нужно получить от юриста

1. **Лицензионный договор / публичная оферта B2B** Podocare ↔ студия (лицензиат).
2. **Приложения:** Тариф v0.3, описание трека публикации A или B, акт приёмки онбординга.
3. **Соглашение об обработке ПДн (DPA)** или раздел в договоре: оператор — заказчик, обработчик — Podocare.
4. **Рекомендации по B2C** для заказчика: что должно быть в политике конфиденциальности и оферте **клиентам студии** (Podocare не сторона B2C).
5. **Шаблоны** (с плейсхолдерами бренда заказчика):
   - согласие на обработку ПДн (мобильное приложение);
   - информированное согласие на процедуры / мед. информацию;
   - ссылки на политику заказчика.

## Продукт и данные (для 152-ФЗ)

**Podocare** — облачная платформа (API, веб-админка, white-label мобильное приложение). Данные клиентов студии хранятся в tenant, изолированном по `networkId`.

### Категории данных клиентов (B2C конечных пользователей)

| Категория | Примеры в системе |
|-----------|-------------------|
| Идентификация | телефон (OTP), ФИО, email |
| Записи и услуги | дата/время приёма, студия, специалист, услуга |
| Здоровье / процедуры | протокол визита, план лечения, ответы квиза, загрузки (фото) |
| Платежи | заказы, статусы оплат (ключи эквайринга — у заказчика) |
| Технические | push-токен, device id, IP (логи), cookie/сессия |
| Маркетинг | предпочтения уведомлений, воронка (funnel events) |

### Типы согласий в приложении (enum в БД)

- `PERSONAL_DATA` — обработка ПДн  
- `MEDICAL_INFORMATION` — мед. информация / процедуры  
- `PROCEDURE_CONSENT` — согласие на процедуру  
- `MARKETING_COMMUNICATIONS` — маркетинг  
- `TERMS_OF_SERVICE` — пользовательское соглашение  

**Важно для юриста:** тексты в коде сейчас содержат бренд «Solodova Recovery System» — для SaaS их нужно **параметризовать под заказчика** (оператор = студия). Юристу дать задачу: шаблон с полями `{STUDIO_LEGAL_NAME}`, `{PRIVACY_POLICY_URL}`, `{SUPPORT_EMAIL}`.

### Роли по 152-ФЗ (позиция Podocare)

| Сторона | Роль | Обоснование |
|---------|------|-------------|
| **Студия (заказчик)** | Оператор ПДн клиентов | Оказывает услуги клиенту, определяет цели обработки |
| **Podocare** | Обработчик по поручению | Хостинг, ПО, хранение в tenant по договору |

### Инфраструктура (уточнить у юриста формулировку)

- Планируется хостинг **в РФ** (VPS / managed Postgres в РФ). Указать в DPA субпроцессоров: SMS-провайдер, Expo Push, ЮKassa/Т‑Банк (если применимо), object storage.
- Трансграничная передача: push через Expo — **вопрос юристу**.

## Два трека публикации мобильного приложения

В договоре **обязательно** выбрать и описать один трек (см. бриф § «Два трека»):

- **Трек A** — Podocare загружает в сторы по поручению заказчика (владелец listing — заказчик).
- **Трек B** — Podocare передаёт сборку; заказчик публикует сам.

## SLA (предложение для договора)

| Параметр | «Старт» | «Сеть» |
|----------|---------|--------|
| Канал | email | email + Telegram |
| Срок ответа | 3 рабочих дня | 24 ч в рабочее время 10:00–19:00 МСК |
| Доступность платформы (цель) | **99%** / календарный месяц (prod) — **согласовать с юристом** |
| Потолок ответственности | сумма подписки за **3** месяца — **согласовать с юристом** |

Исключения из SLA: плановые работы (уведомление за 24 ч), форс-мажор, Apple/Google/SMS/эквайринг.
